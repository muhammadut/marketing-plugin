# Eyal — Hooked

## One-line summary
Eyal's Hook Model is a four-stage cycle (Trigger → Action → Variable Reward → Investment) that habit-forming products run users through repeatedly, and the framework is the most useful scaffolding the lifecycle sub-vertical has for thinking about retention as a process to be designed rather than a metric to be observed.

## Why this tier
Scaffolding, with caveats. Eyal draws on real behavioral science (BJ Fogg's Behavior Model, Skinner's variable-reinforcement work, dopamine prediction-error research from Schultz et al.) — so the *underlying* psychology has empirical grounding. But the **Hook Model itself**, as a four-step procedure, is Eyal's synthesis, not a tested framework with measured effect sizes for each stage. It is a forcing function on lifecycle designers to think about each stage of the loop deliberately. There is also a publicly-debated ethical dimension: Eyal's later book (*Indistractable*) walks back some of the "engineer compulsive use" framing that *Hooked* leans into. Cite the model for design discipline; do not cite it as license to maximize compulsion.

## Key concepts
- **Trigger.** External (notification, email, ad) or internal (boredom, loneliness, anxiety). The job of an external trigger is to get the user to take the action enough times that an internal trigger forms — at which point the product no longer needs to nudge.
- **Action.** The simplest behavior done in anticipation of a reward (open the app, scroll the feed, post a photo). Eyal cites Fogg's B = MAT framing: behavior happens when motivation, ability, and trigger converge. Make the action easy (high ability) and the motivation requirement drops.
- **Variable Reward.** The unpredictable payoff that follows the action. Eyal divides into rewards of the tribe (social validation), rewards of the hunt (information / resources), and rewards of the self (mastery, completion). Variable schedules drive stronger habit formation than fixed ones — this is the Skinner finding the Hook Model rests on.
- **Investment.** The user puts something into the product (data, content, friends, customization, time) that increases future value to them and increases switching cost. Investment loads the next trigger.
- **Habit testing.** Eyal's framework for measuring whether a product has actually formed habits: identify the habit (action and frequency), codify the user segments showing the behavior, modify the product to push more users into the habit segment.
- **The B=MAT model from BJ Fogg.** Behavior = Motivation × Ability × Trigger. The lifecycle-lead can borrow this directly: when a user isn't taking a desired action, diagnose which of the three is missing rather than just "send another email."
- **The ethical caveat.** Eyal himself has subsequently emphasized that the Hook Model should be used to build habits the user *wants* to form, not compulsions the user would reject if they understood them.

## When to cite
Cited by `lifecycle-lead` in `retention-flows.md` (the four-stage loop is the structural template for any habit-forming retention flow) and in `onboarding-flow.md` (the B=MAT model is the diagnostic for stuck users). Cited by `positioning-lead` when reasoning about whether the product is positioned around a behavior the user actually wants to repeat.

## Anti-citation guidance
Do not cite Hooked as license to maximize compulsion — Eyal himself walked this back in *Indistractable*, and the modern read is that habit formation should serve user goals, not work against them. Do not cite the four-stage model as if it were a research finding with measured effect sizes — it is a synthesis, and the variance from one product to another is enormous. Do not apply the model to one-time-use transactional products (a marketplace booking, an emergency call) where there is no "habit loop" to speak of.
